# address book system
第一次尝试，通讯录系统

项目简介

本项目是基于 C++ 开发的控制台通讯录管理系统，在原有纯内存存储版本基础上，改造升级为 MySQL 数据库持久化存储。实现联系人增、删、改、查、清空等完整业务功能，数据持久化保存，重启程序数据不会丢失。

二、开发遇到的问题及解决方案
1. 程序重启后数据全部丢失
问题现象：关闭重启程序，之前添加的联系人全部消失
原因：最初仅依靠数组做内存存储，无持久化能力
解决：对接 MySQL 数据库，程序启动自动读取库中数据加载到内存

2. 中文存入 MySQL 报错
问题现象：插入中文姓名，数据库报编码错误
原因：VSCode 文件、Windows 终端、MySQL 三者编码不统一，早期习惯使用 GBK 编码
解决：全部统一为 UTF-8；控制台设置 UTF-8 编码；MySQL 连接设置utf8mb4字符集

3. VSCode 右上角运行按钮无法运行项目
问题现象：Ctrl+Shift+B 可正常编译运行，右上角启动按钮直接报错
原因：下载过多vscode插件，如Code Runner,C/C++ Runner,C/C++(Microsoft)等多个独立可进行cpp的编译系统，造成配置混乱，运行按钮不会读取配置好的编译参数，无法识别 MySQL 头文件与库文件
解决：统一使用 Ctrl+Shift+B 编译 + 终端运行 的方式（Microsoft）或删除不必要插件

4. SQL 语句执行成功，数据库无数据
问题现象：控制台提示添加成功，数据表无新增数据
原因：SQL 语句引号书写格式错误，语句执行实际失败
解决：规范 SQL 拼接格式，增加执行结果判断，打印错误日志排查

四、项目克隆后配置部署教程
克隆项目
bash
git clone 本项目仓库地址

数据库初始化
打开 MySQL，新建数据库：address_book
执行建表语句：尽量不要修改表名
sql
CREATE TABLE IF NOT EXISTS contact (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50) NOT NULL,
    sex INT,
    age INT,
    phone VARCHAR(20),
    addr VARCHAR(100)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
修改数据库连接配置
在系统中设置环境变量 `MYSQL_PASSWORD`，值为你的MySQL密码：
    - Windows: 打开命令提示符，输入 `set MYSQL_PASSWORD=你的密码`
    - Linux/macOS: 打开终端，输入 `export MYSQL_PASSWORD=你的密码`
如需修改数据库地址、用户名、数据库名或端口，可在 `main.cpp` 开头的配置常量中修改。

项目编译运行
VSCode 中按下 Ctrl + Shift + B 完成编译
终端进入项目 src 目录，执行运行命令：

## 项目更新日志
- [2026-05-24] 本次迭代：
  - 修复了新增联系人自增ID同步问题，当前会话内数据读写完全一致
    问题现象：新增联系人后，内存中 ID 为0，仅重启程序后才能从数据库读取正确 ID，导致当前会话内修改 / 删除操作无法定位数据。
    • 解决方案：
    ◦ 调整代码执行顺序：先执行INSERT语句写入数据库，再通过mysql_insert_id()获取 MySQL 生成的自增 ID，最后将更新了 ID 的对象存入内存数组。
    ◦ 确保当前会话内新增联系人的 ID 与数据库完全一致，修改、删除功能可直接通过真实 ID 定位，无需重启程序。

  - 完善了全字段输入校验，提升数据规范性
    为所有字段补充了合法性校验，包括：
    ◦姓名非空校验，禁止录入空白联系人。
    ◦手机号格式校验（11 位纯数字、1 开头），避免无效号码存入数据库。
    ◦性别 / 年龄取值范围校验，过滤非法输入。
    ◦地址空值处理，空字符串自动转换为 SQL 的NULL，适配数据库设计。

  - 优化数据库建表脚本，添加字段约束与注释
    重构address_book_init.sql，为字段添加必要约束：
    ◦phone字段设置NOT NULL UNIQUE，保证手机号非空且不重复。
    ◦统一字符集为utf8mb4，解决中文乱码问题。
    ◦为sex/age/addr字段添加NULL支持，适配 “未填写” 场景。
    ◦增加字段注释，提升脚本可读性与维护性。

  - 优化空值处理逻辑，终端显示更友好
    完善了数据库读取逻辑，NULL值自动转换为程序默认值：
    ◦sex/age字段的NULL值转换为-1，addr字段的NULL值转换为空字符串。
    ◦终端显示时，-1和空字符串统一显示为 “未填写”，提升用户体验。
    规范了 SQL 拼接逻辑，字符串字段自动添加引号，避免语法错误。
